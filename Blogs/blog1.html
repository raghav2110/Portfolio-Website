Execution Context In Javascript

In this blog we will learn how Javascript works. Do you know what is Execution Context?

Execution Context is

  • It is a concept that hold info about the environment within which current code is being executed.
  • Imagine it as a closed container inside which Javascript runs
  • Execution Context is a place where all the things happening in Javascript takes place.

The container is divided into 2 components. The first component is Memory Component and the second component is Code Component. The memory component stores all the variables and function. All the values are stored in the form of key value pair. It is also called as Variable Environment. The code component is the space where code is executed line by line. It is also called as Thread Of Execution.

There Are Three Types Of Execution Context:

  • Global Execution Context
  • Functional Execution Context
  • Eval Function Execution Context

All the execution stack that are created during the code execution are stored in a Execution Stack also called as "Calling Stack". The stack uses LIFO structure for the execution. In Javascript engine creates a global execution context and pushes variables and function on the top of the stack. The engine then start execution from the top of the stack, once the execution is complete it pops it from the stack and control is passed to next context.

So this was some basic about Execution Context and how Javascript works.
